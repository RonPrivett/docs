You have two deployment options with BigAnimal: 

- Deploy clusters in your own cloud account with a cloud service provider
- Deploy clusters in BigAnimal's cloud account

To learn about the pricing of these two deployment options, see [Pricing and billing](/biganimal/latest/pricing_and_billing/).

## Using your own cloud account

When deploying in your own cloud account, you need to set up your cloud service provider. You can use your AWS, Azure, or Google Cloud account. In this deployment model, you are responsible for your managing your cloud resources and connecting your cloud to BigAnimal. 

For a list of topics specific to your chosen cloud account, see:

- [Deploying with Azure](deploying_azure)
- [Deploying with AWS](deploying_aws)
- [Deploying with Google Cloud](deploying_gcp)

## Using BigAnimal's cloud account

BigAnimal's cloud account offers a seamless deployment option if you don't want to set up a separate cloud account for your clusters. You can deploy a cluster in BigAnimal's cloud account instantly. 

For a list of topics specific to BigAnimal's cloud account, see [Deploying with BigAnimal's cloud](deploying_bah).

!!! Note Note
Currently, when you deploy in BigAnimal's cloud account, you can use AWS or GCP as your cloud provider.
!!!
